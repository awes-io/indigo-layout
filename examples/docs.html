Предназначен для отображения скрытого по-умолчанию списка нажатием на кнопку

Минимальная разметка

Компонент состоит из раскрывающейся обёртки (dropdown) и элементов. Без элементов внутри обёртки, она не имеет смысла, поэтому минимальная разметка выглядит как

<context-menu>
    <cm-button>Click me!</cm-button>
</context-menu>

Внешний вид

Click me!

Сгенерированная разметка

<div class="context-menu">
    <button aria-haspopup="true" class="context-menu__toggler context-menu__toggler-default">
        <i class="icon icon-dots"></i>
    </button>
    <div aria-expanded="false" class="context-menu__dropdown" style="display: none;">
        <ul class="context-menu__list">
            <li class="cm-item">
                <button class="cm-item__button">Click me!</button>
            </li>
        </ul>
    </div>
</div>

Открытие выпадающего списка происходит нажатием на открывающую кнопку, закрытие кликом на ту же кнопку, на ссылку или кнопку среди элементов контекстного меню или кликом снаружи выпадающего меню

Параметры компонента-обёртки <context-menu>

Название Тип По умолчанию Опиание
button-class String undefined Дополнительный класс для кнопки, открывающей список
right Boolean false Приоритет в расположении списка по правой границе открывающей кнопки
top Boolean false Приоритет в раскрытии списка вверх, располагаясь сверху открывающей кнопки
auto-position Boolean true Автоматическое позиционирование списка в зависимости от заданных границ и видимости на экране
boundary String body Селектор элемента - границы для автоматического позиционирования списка

Позиционирование на экране

Раскрывающийся список (dropdown) располагается относительно открывающей кнопки, и по-умолчанию примыкает к нижней и левой границам и открывается вниз

Автоматическое позиционирование: Заданное расположение является лишь рекомендацией, если включен параметр auto-position = true. Если при заданном положении раскрывающийся список не отображается в области видимости или не вписывается в ограничительный контейнер, то он изменит своё положение, так чтобы:

  1. отобразиться на экране максимально видимым
  2. вписаться в границы

Ограничительный контейнер: Свойство boundary - селектор границы, по-умолчанию, или если по-селектору не найден элемент среди родителей компонента этим контейнером является контейнер страницы body

Выпадающий список не может раскрыться вниз и вправо, но может вписаться в границы поэтому он открывается вверх и влево

Элементы контекстного меню

Кнопка <cm-button>

<context-menu>
    <cm-button @click="someMethod">Click me!</cm-button>
</context-menu>

Свойства элемента <cm-button>

Название Тип По умолчанию Опиание
itemClass String undefined Дополнительный класс для кнопки

На элемент нужно навешивать обработчики событий и можно добавлять дополтительные атрибуты. Без бработчика клик по кнопке будет просто закрывать открытое меню. Такая разметка

<cm-button class="list-item-class" item-class="item-class" @click="$notify({group: 'right-b', type: 'sucсess', title: 'Notification!'})"> Send notification </cm-button>
Send notification

Будет сгенерирована в такой HTML, по клику будет вызван метод someMethod:

<li class="cm-item list-item-class">
    <button aria-label="Awesome Button!" class="cm-item__button item-class">
        Send notification
    </button>
</li>

Ссылка <cm-link>

<context-menu>
    <cm-link href="https://some.url">This is a link elemen</cm-link>
</context-menu>
This is a link element

Свойства элемента <cm-link>

Название Тип По умолчанию Опиание
itemClass String undefined Дополнительный класс для ссылки
href* String undefined url или якорь, обязательное свойство

Также на элемент можно навешивать обработчики событий и добавлять дополтительные атрибуты, для примера, такая разметка

<context-menu>
    <cm-link href="#anchor"
            class="list-item-class"
            item-class="item-class"
            data-prop="someProp"
            @click.prevent="someMethod">
        This is a link element
    </cm-link>
</context-menu>
This is a link element

Будет сгенерирована в такой HTML, по клику перехода не будет, а будет вызван метод someMethod:

<li class="cm-item list-item-class">
    <a href="#anchor" data-prop="someProp" class="cm-item__link item-class">
        This is a link element
    </a>
</li>

Разделитель <cm-separator>

<context-menu>
    <cm-button>Above separator</cm-button>
    <cm-separator></cm-separator><!-- разделитель -->
    <cm-button>Below separator</cm-button>
</context-menu>
Above separator Below separator

Панель <cm-panel>

Компонент предназначен для размещения контента внутри контекстного меню

<context-menu>
    <cm-panel>
        Default panel
    </cm-panel>
    <cm-panel secondary>
        Secondary panel
    </cm-panel>
</context-menu>
Default panel Secondary panel

Свойства элемента <cm-panel>

Название Тип По умолчанию Опиание
secondary Boolean false Меняет цвет фона на более тёмный

Сгенерированная разметка

<li class="cm-item">
    <div class="cm-item__panel">
        Default panel
    </div>
</li>
<li class="cm-item">
    <div class="cm-item__panel is-secondary">
        Secondary panel
    </div>
</li>

GET-параметры в адресной строке <cm-query>

Компонент предназначен для вставки GET-параметров в адресную строку браузера без перезагрузки (например для фильтрации или сортировки), другие компоненты могут отслеживать эти параметры и обновлять данные.

Свойства элемента <cm-query>

Название Тип По умолчанию Опиание
itemClass String undefined Дополнительный класс для кнопки
param* Object undefined Обязательное свойство. Параметры, которые необходимо подставить или заменить в адресной строке, и при совпадении с которыми кнопка полузит активное состояние 
<context-menu>
    <cm-query :param="{state: 'active'}">
        I am active
    </cm-query>
    <cm-query :param="{state: 'not-active'}">
        I am not act
</context-menu>
I am active I am not active

HTML выглядит как

<li class="cm-item">
    <button class="cm-item__button is-active">
        I am active
    </button>
</li>
<li class="cm-item">
    <button class="cm-item__button">
        I am not active
    </button>
</li>

AJAX-запрос <cm-ajax>

Компонент предназначен для отправки данных на сервер с возможностью вызова метода для обработки полученного ответа, обрабатывая событие sended, в который передаются данные ответа. Для отправки запроса используется миксин serverRequest

Свойства элемента cm-ajax

Название Тип По умолчанию Опиание
itemClass String undefined Дополнительный класс для кнопки
params Object {} Объект данных для отправки
url* String undefined Обязательный параметр. Адрес запроса
method String get Метод для проведения запроса, ограничен вариантами get, put,post, delete, patch

События, вызываемые элементом cm-ajax

Название Аргументы Опиание
sended data (Object) Вызвается после получения ответа и передаёт присланные данные 
<context-menu>
    <cm-ajax
        method="post"
        url="https://httpbin.org/post"
        :params="{param1: 'one', param2: 'two'}"
        @sended="({ data }) => $notify({ group:'right-b', text: JSON.stringify(data) })">
        Send request
    </cm-ajax>
</context-menu>
Send request

Разультат в HTML

<li class="cm-item">
    <button class="cm-item__button">
        Send request
    </button>
</li>

Стилизация открывающей кнопки

Кнопка, открывающая контекстное меню может быть изменена с помощью слота, с именем toggler. Содержимое слота помещается внутри тега, поэтому не должно содержать блочных по семантике элементов (<div>). Блочный контент должен быть представлен в виде 11<span style='display:block;' >

Пример

<context-menu>
    <span slot="toggler" class="new-toggler">Open me</span>
    <cm-button>
        Click me!
    </cm-button>
</context-menu>

<style>
.new-toggler {
    display: inline-block;
    color: white;
    background-color: lightseagreen;
    border: 1px solid aqua;
    padding: 5px 15px;
    margin-bottom: 3px;
    border-radius: 15px;
    height: 30px;
}
</style>
Open me Click me!
